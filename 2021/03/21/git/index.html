git使用

发表于 2021-03-21 更新于 2021-03-26 阅读次数：

git init 初始化

git status 查看文件状态

git add . 追踪文件

git commit -m 提示信息，向仓库提交代码

git log 查看提交记录

若按照以下顺序

  1. git init  初始化本地仓库
  2. git remote add origin 地址   添加远程仓库地址
  3. 如果你在这之后就执行
  4. git add -A，
  5. git commit -m “ “
  6. git push origin master, 那么就会出现这个问题(被拒绝)，
    ! [rejected] master -> master (non-fast-forward) error: failed to push some refs to 'c-plus-plus'

所以在remote add后不要着急git add，一定要git pull origin master，出现这个原因

即码云上初始化有readme文件而本地没有，故本地与云端不同步

方法：先拉再推 先pull 再push

​ git push -f origin master 强制上传（不建议）
